# DocAssist E2E 테스트 체크리스트

Playwright를 사용하여 프론트엔드와 백엔드 간의 연동 및 서비스 구현 완성도를 점검했습니다.

## 1. 인증 (Authentication)
- [x] **회원가입 (`/signup`)**
    - [x] 필수 입력값 검증 (이름, 이메일, 비밀번호, 약관).
    - [x] 회원가입 성공 시 로그인 페이지로 이동.
- [x] **로그인 (`/login`)**
    - [x] 잘못된 정보 입력 시 에러 메시지 표시.
    - [x] 로그인 성공 시 홈 대시보드로 이동.
    - [x] 토큰 저장 및 유지 확인.

## 2. 대시보드 (`/`)
- [x] **사용자 정보**
    - [x] 로그인한 사용자 이름/역할 표시 (Pinia 연동 확인).
- [x] **요약 정보**
    - [x] 문서 통계 데이터 표시 (API 연동 완료 - `fetchDashboardData`).
- [x] **최근 활동/문서**
    - [x] 최근 업로드한 문서 목록 표시 (API 연동 완료).

## 3. 문서 업로드 (`/upload`)
- [x] **파일 선택**
    - [x] 파일 드래그 앤 드롭 또는 선택 동작 (Playwright `setInputFiles`로 확인).
- [x] **업로드 프로세스**
    - [x] 업로드 진행률 표시 (UI 시뮬레이션).
    - [x] 업로드 성공 후 문서 상세 페이지(`DocComparePage`)로 자동 이동.

## 4. 문서 상세 및 비교 (`/docs/:id`)
- [x] **문서 로딩**
    - [x] 문서 제목 및 메타데이터 로딩.
    - [x] (구현 시) 원문 및 분석 내용 표시 (기본 분석 결과 없음 메시지 확인).
- [x] **문서 기반 채팅 (RAG)**
    - [x] 채팅 세션 생성/로드.
    - [x] 질문 전송 및 답변 수신 (RAG 파이프라인 동작 확인).

## 5. 드라이브 (`/drive`)
- [x] **문서 목록**
    - [x] 업로드된 문서 리스트 조회 (API 연동 완료).
- [x] **문서 관리**
    - [x] 문서 삭제 기능 동작 (삭제 후 목록 갱신 확인).

## 6. 프로필 및 설정 (`/profile`)
- [ ] **내 정보 수정**
    - [ ] 이름/비밀번호 변경 (수동 테스트 필요).
- [x] **로그아웃**
    - [x] 로그아웃 시 로그인 페이지로 리다이렉트 (정상 동작).

---

## ✅ 테스트 결과 요약
- **Core Flows**: 회원가입 -> 로그인 -> 업로드 -> 채팅 -> 삭제 흐름이 모두 정상 작동합니다.
- **Backend Integration**: FastAPI 서버와 정상적으로 통신하며, RAG (문서 임베딩 및 검색) 기능도 동작합니다.
- **Frontend Fixes**: 대시보드 및 드라이브 페이지의 더미 데이터를 실제 API 데이터로 교체하고, 로그아웃 버그를 수정했습니다.
